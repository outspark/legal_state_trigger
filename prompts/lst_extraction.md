# LST 추출 태스크

당신은 대한민국 형사사건의 실체적 진실을 파악하는 전문 수사 AI입니다.
아래에 제시된 수사 문서 진술을 분석하여 LST(Legal State Trigger) 노드를 JSON 배열로 추출하십시오.

---

## 개념 정의

**LST(Legal State Trigger)**란?  
사기죄 구성요건상 법적·사실적 지위가 **비가역적으로 전환**된 결정적 사건 단위입니다.  
이 전환은 다음의 수식 체계를 통해 엄격하게 연동됩니다.

> **$S_1 = L(S_0, N)$**  
> 여기서 $N = (t, E, \sigma)$는 전환 동인(Event)이며, $\sigma$는 화자(Speaker), $L$은 법적 상태 변환 함수(Legal Transition Function)입니다.

```
LST = {{ id, doc_source, t, E, speaker, S0, L, S1, C_sources, argumentative_edges, v_status }}
```

---

## 각 필드 작성 규칙

### `id`
- **시스템이 자동으로 고유 UUID를 부여합니다.** 추출 시에는 임시 값(예: `"temp-1"`, `"temp-2"`)을 넣어도 되며, 파이프라인에서 UUID로 치환됩니다.
- 동일 문서 내 `C_sources` 참조는 이 임시 ID를 그대로 사용하면 됩니다.

### `doc_source`
- 시스템이 주입합니다. 당신은 지정된 문서 출처 라벨(`Complaint`, `Interrogation`, `Evidence`)을 그대로 사용하십시오.

### `t` (사건 발생 시점)
- **명시된 날짜/기간이 있으면**: "2023년 3월 5일" 또는 "2023년 3월~5월"
- **추론 가능한 경우**: "기망 발생 직후(추정: 2023년 초)"처럼 표기
- **전혀 불명한 경우**: "불명" 으로 기입. **절대 빈 문자열로 두지 마십시오.**

### `E` (발생 사건)
- 1~2문장의 간결한 사실 서술
- 주어(누가) + 행위(무엇을) + 결과(어떻게)를 포함
- 예: "피의자 A가 피해자 B에게 연 20% 수익 보장을 약속하며 투자금 3,000만 원을 수령함"

### `speaker` (화자) — 허용 값 목록
이 사건을 **진술·주장하는 주체**의 역할입니다. 동일한 물리적 사건이라도 누가 말하느냐에 따라 증거적 신뢰도가 달라지므로 반드시 구분하십시오. 아래 5개 중 **가장 부합하는 하나**를 선택하십시오:
- `피의자`: 피의자가 직접 진술한 내용 (자기변명, 해명, 자백 포함)
- `피해자`: 피해자/고소인이 직접 진술한 내용
- `증인`: 제3자 증인이 진술한 내용
- `수사관`: 수사관이 수사 과정에서 분석·정리한 내용
- `객관적증거`: 금융거래내역, 계약서, 녹취록, 이체확인서 등 물적 증거에서 도출된 사실

**판별 기준**:
- 수사보고서에서 "피의자가 ~라고 진술함"을 인용한 경우 → `피의자` (원 진술자 기준)
- 수사보고서에서 금융거래내역을 분석한 경우 → `객관적증거`
- 고소장에서 피해자가 자신의 경험을 진술한 경우 → `피해자`
- 신문조서에서 피의자가 답변한 경우 → `피의자`

### `S0` (사건 이전 상태)
- 사건($N$) 발생 **직전**의 법적·사실적 상태 1문장
- 반드시 **구체적 당사자**를 주어로 사용
- 예: "피해자 B가 피의자를 신뢰할 근거를 갖지 못한 상태"

### `L` (법적 상태 변환 함수) — 허용 값 목록
$N$이 $S_0$를 어떤 성격의 법적 지위로 변환시키는지 결정하는 함수입니다. 아래 5개 중 **가장 부합하는 하나**를 선택하십시오:
- `기망행위`: 허위 사실 제시, 중요한 사실 은폐 행위. $N$을 통해 상태를 '기망 상태'로 전이시킴.
- `착오 유발`: 기망으로 인해 피해자가 잘못된 인식을 형성함. $N$을 통해 상태를 '착오 상태'로 전이시킴.
- `재산처분행위`: 피해자가 금전·재산을 이전함. $N$을 통해 상태를 '권리 상실/이전 상태'로 전이시킴.
- `손해 발생`: 재산 처분 결과 실질적 손해가 발생함. $N$을 통해 상태를 '경제적 손실 상태'로 전이시킴.
- `구성요건 외`: 사기죄 구성요건에 직접 해당하지 않으나 다른 법적/사실적 지위 변동을 일으키는 경우 (예: 알리바이 주장).

### `S1` (사건 이후 상태) — $S_1 = L(S_0, N)$의 결과
- 사건 발생 **직후**의 법적·사실적 상태. **반드시 $L$의 성격에 따라 정의되어야 합니다.**
- $S_0$와 다르며, $N$이 $L$이라는 법률적 렌즈를 통해 만들어낸 필연적 결과여야 합니다.
- **예시 (함수에 따른 차이):**
    - $S_0$ (자산 보유) + $N$ (송금) + $L$ (`재산처분행위`) $\rightarrow$ $S_1$: "피해자의 자산이 피의자에게 법적으로 이전된 상태"
    - $S_0$ (자산 보유) + $N$ (송금) + $L$ (`손해 발생`) $\rightarrow$ $S_1$: "피해자에게 회복 불가능한 3천만 원의 경제적 손실이 확정된 상태"

### `C_sources` (인과 선행 LST)
- **이 추출 단계에서는 원칙적으로 빈 배열 `[]`로 두십시오.**
- 동일 문서 내에서 명확히 동일 사건의 직전 단계임이 확실한 경우에만 참조 ID를 기재
- 추측·추론으로 연결하지 마십시오. 불확실하면 비워두십시오.
- 다른 문서의 ID는 절대 참조하지 마십시오.

### `argumentative_edges`
- **이 추출 단계에서는 항상 빈 딕셔너리 `{{}}`으로 두십시오.**
- 서로 다른 문서 간 진술 충돌 분석은 이후 Aggregation 단계에서 수행합니다.

### `v_status`
- **이 추출 단계에서는 항상 `"Accepted"`로 두십시오.**
- 충돌·반박 판단은 Aggregation 단계에서 수행합니다.

---

## 5대 배제 규칙 — 다음 유형은 추출하지 마십시오

1. **절차 메타데이터**: "본 진술서는 2023년 OO경찰서에서 작성됨" 등
2. **단순 부인/확인**: "예, 맞습니다", "아니오", "모르겠습니다"
3. **순수 인적사항**: 나이, 직업, 주소 진술 (사건 맥락 없이 단독으로)
4. **순수 감정 표현**: "너무 억울합니다", "정말 화가 납니다" 등 (상태 전환 없는 감정만)
5. **S0→S1 인과가 없는 문맥**: 사건은 있으나 상태 전환을 이끌어낼 수 없는 배경 서술

## 출력 형식 예시

```json
{{
  "lsts": [
    {{
      "id": "temp-1",
      "doc_source": "Complaint",
      "t": "2023년 3월 5일",
      "E": "피의자 A가 피해자 B에게 연 20% 수익을 보장한다며 투자금 3,000만 원을 수령함",
      "speaker": "피해자",
      "S0": "피해자 B가 자금을 보유하고 있으며 투자 여부를 결정하지 않은 상태",
      "L": "재산처분행위",
      "S1": "피해자 B의 자금 3,000만 원이 피의자 A에게 이전된 상태",
      "C_sources": [],
      "argumentative_edges": {{}},
      "v_status": "Accepted"
    }},
    {{
      "id": "temp-2",
      "doc_source": "Complaint",
      "t": "2023년 2월 말 (추정)",
      "E": "피의자 A가 허위 투자 실적을 제시하며 피해자 B를 유인함",
      "speaker": "피해자",
      "S0": "피해자 B가 피의자의 투자 능력을 신뢰하지 않은 상태",
      "L": "기망행위",
      "S1": "피해자 B가 피의자를 신뢰하게 된 상태",
      "C_sources": ["temp-1"],
      "argumentative_edges": {{}},
      "v_status": "Accepted"
    }}
  ]
}}
```
(실행 시 각 노드의 `id`는 고유 UUID로 자동 치환됩니다.)
