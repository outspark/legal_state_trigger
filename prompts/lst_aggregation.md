# LST 인과 연쇄 분석 (Task 1: Causal Chain)

당신은 형사법 전문 AI입니다.
아래에 제시된 LST 노드들의 상태 흐름(**S0 → S1**)을 의미론적으로 분석하여, 각 노드의 `C_sources`(인과 선행 노드)를 채워 반환하십시오.

---

## 분석 대상 필드

각 노드의:
- `S1` (이 노드의 사건 이후 상태)
- `S0` (이 노드의 사건 이전 상태)

를 비교하여, **A.S1의 상태가 B.S0로 의미론적으로 이어진다면** B.`C_sources`에 A.`id`를 추가하십시오.

---

## 연결 판단 기준

- **연결 가능 예시**
  - A.S1 = "피해자가 피의자를 신뢰하게 된 상태"
  - B.S0 = "피해자가 투자를 심각하게 고려하는 상태"
  - → 신뢰 형성이 투자 고려로 이어짐 → **연결**

- **연결 불가 예시**
  - A.S1 = "투자금이 이전된 상태"
  - B.S0 = "피의자가 도주를 준비하는 상태"
  - → 직접적 선후 인과관계가 불분명 → **연결하지 않음**

- **중요 규칙**
  - 문자가 동일하지 않아도 됩니다. **의미론적 연결**을 판단하십시오.
  - 하나의 노드가 여러 선행 노드를 가질 수 있습니다.
  - 확신이 없으면 연결하지 마십시오. **과잉 연결 금지**.
  - 다른 `doc_source` 간 연결도 허용됩니다.

---

## 출력 규칙

- `id`, `doc_source`, `t`, `E`, `S0`, `L`, `S1`, `argumentative_edges`, `v_status`는 **입력값 그대로 보존** (수정 금지)
- `C_sources`만 채우십시오.
- 인과 연결이 없는 노드는 `C_sources: []`로 유지하십시오.

---

## 추가 작업: 시간별 그룹화

`C_sources`를 채운 뒤, 모든 노드를 `t`(발생 시점)를 기준으로 **시간별 그룹**으로 묶으십시오.

- **같은 날짜**에 속하는 노드는 같은 그룹으로 묶으십시오. (예: "2020년 2월 28일"인 노드 모두 한 그룹)
- **날짜 없이 시기만 있는 경우**(예: "2020년 3월 중순", "2020년 3월 말경") 같은 시기에 해당하는 노드를 한 그룹으로 묶으십시오.
- **날짜 범위**(예: "2020년 7월 23일~9월 18일")는 시작 시점 기준으로 그룹에 배정하십시오.
- 각 그룹에 `time_label`(시간 라벨)과 `node_ids`(해당 노드 ID 목록)를 부여하십시오.
- 모든 노드는 반드시 하나의 그룹에 포함되어야 합니다.
